💡 WAF מבין את תוכן הבקשה, FW רק את הכותרות

Kerberos - פרוטוקול AD

פרוטוקול ברירת מחדל ל-Active Directory

תהליך האימות:

  1. AS-REQ → User שולח בקשה ל-KDC
  2. AS-REP → KDC מחזיר TGT (Ticket Granting Ticket)
  3. TGS-REQ → User מבקש גישה לשירות עם TGT
  4. TGS-REP → KDC מחזיר Service Ticket
  5. AP-REQ → User מציג ticket לשירות
  6. AP-REP → השירות מאשר גישה

🔑 Port: 88 (TCP/UDP)

LDAP

Lightweight Directory Access Protocol

  • פרוטוקול לגישה לשירותי directory
  • Port 389 (LDAP) / 636 (LDAPS - מוצפן)
  • משמש לשאילתות על users, groups, computers
  • מבנה היררכי: DC → OU → Objects

⚠️ LDAP Injection - התקפה דרך input לא מסונן

IOC vs IOA

IOC - Indicator of Compromise

  • עדות לפריצה שכבר התרחשה
  • דוגמאות: Hash זדוני, IP חשוד, domain מזיק
  • רטרואקטיבי - מזהה אחרי האירוע

IOA - Indicator of Attack

  • התנהגות שמעידה על תקיפה פעילה
  • דוגמאות: ריבוי login failures, lateral movement
  • פרואקטיבי - מזהה בזמן אמת

בדיקת התנהגות קובץ

Static Analysis:

  • בדיקת Hash (VirusTotal, etc.)
  • ניתוח strings בקובץ
  • בדיקת metadata וחתימות

Dynamic Analysis:

  • הרצה ב-Sandbox (Any.run, Joe Sandbox)
  • ניטור תקשורת רשת
  • מעקב אחר שינויים ב-registry/files
  • בדיקת processes שנוצרים

DCSync Attack

מה זה? תוקף מתחזה ל-DC ומבקש סנכרון סיסמאות

🚨 חשוד במיוחד:

  • DCSync לא בין Domain Controllers
  • בשעות חריגות (23:00)
  • ממחשב שאינו DC

Detection:

  • Event ID 4662 - Directory Service Access
  • חיפוש Replication rights מ-non-DC

LOLBins / LOLBas

Living Off The Land Binaries

כלים לגיטימיים של Windows שתוקפים מנצלים:

  • PowerShell - הרצת סקריפטים, download & execute
  • certutil - הורדת קבצים, decode base64
  • WMI - הרצה מרחוק, persistence
  • mshta - הרצת HTA files
  • regsvr32 - הרצת COM objects
  • rundll32 - הרצת DLLs

🔗 lolbas-project.github.io

חוקי SIEM מומלצים

  • 🔐 Multiple failed logins → Brute force
  • 👤 Admin login בשעות חריגות
  • 🌍 Login מ-GeoIP חריג
  • 📁 Mass file access/deletion
  • 🔄 DCSync from non-DC
  • ⬇️ certutil download activity
  • 💻 PowerShell encoded commands
  • 🔍 Port scanning activity
  • 🚪 Lateral movement patterns
  • 📧 Email forwarding rules created

SSRF

Server-Side Request Forgery

תוקף גורם לשרת לשלוח בקשות בשמו

סיכונים:

  • גישה לשירותים פנימיים
  • קריאת metadata (AWS 169.254.169.254)
  • סריקת רשת פנימית
  • עקיפת firewall

הגנה:

  • Whitelist של URLs מותרים
  • חסימת internal IPs
  • אימות input

Password Never Expires

למה בעייתי?

  • ⏰ יותר זמן לתוקף לפצח
  • 🔄 לא מאלץ שינוי אחרי דליפה
  • 📋 לרוב מעיד על service account לא מנוהל
  • 🎯 יעד מועדף לתוקפים

מתי לגיטימי?

  • Service accounts עם סיסמאות חזקות מאוד
  • Managed Service Accounts (gMSA)

Phishing / Social Engineering

סוגי פישינג:

  • Spear Phishing - ממוקד לאדם/ארגון
  • Whaling - מכוון להנהלה בכירה
  • Vishing - פישינג טלפוני
  • Smishing - פישינג ב-SMS

סימנים לזיהוי:

  • דחיפות מוגזמת
  • שגיאות כתיב/תחביר
  • Domain דומה אך שונה
  • בקשה למידע רגיש

DMARC / SPF / DKIM

SPF (Sender Policy Framework)

DNS record שמגדיר אילו שרתים מורשים לשלוח מייל מהדומיין

DKIM (DomainKeys Identified Mail)

חתימה דיגיטלית על המייל - מאמת שהמייל לא שונה

DMARC (Domain-based Message Auth)

מדיניות שמגדירה מה לעשות אם SPF/DKIM נכשלים

  • none - רק דיווח
  • quarantine - לספאם
  • reject - לדחות

Port Scan

שלב ראשון ב-reconnaissance

סוגים נפוצים:

  • TCP Connect - חיבור מלא
  • SYN Scan - חצי חיבור (stealth)
  • UDP Scan - לשירותי UDP
  • FIN/XMAS/NULL - עקיפת FW

Detection:

  • הרבה SYN ללא ACK
  • ניסיונות לפורטים סגורים רבים
  • תבנית סריקה סדרתית

Hash

פונקציה חד-כיוונית ליצירת "טביעת אצבע"

שימושים באבטחה:

  • זיהוי קבצים זדוניים (IOC)
  • אימות שלמות קבצים
  • אחסון סיסמאות

אלגוריתמים:

  • MD5 - 128bit (לא מאובטח)
  • SHA-1 - 160bit (לא מומלץ)
  • SHA-256 - 256bit (מומלץ)

CVE

Common Vulnerabilities and Exposures

מזהה אוניברסלי לפגיעויות אבטחה

מבנה:

CVE-YYYY-NNNNN

  • YYYY - שנה
  • NNNNN - מספר סידורי

CVSS Score:

  • 0.0-3.9 - Low
  • 4.0-6.9 - Medium
  • 7.0-8.9 - High
  • 9.0-10.0 - Critical

Security Misconfiguration

סוגים נפוצים:

  • 🔓 Default credentials לא שונו
  • 📂 Directory listing מופעל
  • 🛠️ Debug mode בפרודקשן
  • ☁️ S3 bucket ציבורי
  • 🔑 הרשאות יתר
  • 📡 פורטים מיותרים פתוחים
  • 🔒 הצפנה חלשה או חסרה
  • 📝 Error messages מפורטים מדי

🌙 תרחיש: Admin Login בלילה

משמרת לילה - חיבור אדמין - מה תעשה?

בדיקות ראשוניות:

  1. האם יש תחזוקה מתוכננת?
  2. מאיפה החיבור? (IP, מכונה)
  3. האם ה-user אמיתי או service account?
  4. מה הפעולות שבוצעו?
  5. האם יש baseline לפעילות רגילה?

אם חשוד:

  • 📞 יצירת קשר עם המשתמש/מנהל
  • 🔒 שקול חסימת חשבון
  • 📊 בדיקת לוגים נוספים
  • 🎫 פתיחת ticket והסלמה

🔍 תרחיש: תקשורת חשודה

קובץ לגיטימי שמבצע תקשורת לכתובת AWS נקייה

ניתוח:

  1. מה הקובץ עושה בדיוק?
  2. האם התקשורת צפויה לפי התיעוד?
  3. מתי התחילה התקשורת?
  4. מה נשלח/מתקבל?

נקודות למחשבה:

  • 🌩️ AWS IP יכול להיות C2 לגיטימי-למראה
  • 📦 Supply chain attack?
  • 🔄 DLL hijacking?
  • ⏰ Reputation לא עדכני
